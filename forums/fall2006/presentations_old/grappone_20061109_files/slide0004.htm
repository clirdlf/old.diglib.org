Once we have the content we have a rather large set of content from various sources.  Our current system contains about 300k catalog records, 1 million web pages, about 150k records from the digital archive and about 600 records from 24/7 chat as well as content harvested from places like Amazon. We need a way to organize that content a bit.  We use the Internet Archive's open-source web crawler Heritrix to create compressed archives of the data.  Once we have the dataset from the crawl we use lucine, a text search engine library from Apache to create a full text index.  This index can also be crawled by google and other commercial search engines.   We also have a OAI-PMH compliant set of  Marc records.  One of the added benefits of the system is that we are able to migrate records from Gandhara into the ILS.  For instance records for our electronic thesis and dissertation collection within DSpace can be synchronized with the catalog.
