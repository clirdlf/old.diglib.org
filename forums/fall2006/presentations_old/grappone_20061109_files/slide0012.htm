Wrapper (Gandhara) XML
•<?xml version="1.0"?>
•<!ELEMENT page-list (page*)>      
•<!ELEMENT page(name,title,subject*,author*,description,source,archive,offset,sourceurl,electronicurl,thumbnailurl,collection,content,date)      
•<!ELEMENT name (#PCDATA)>      
•<!ELEMENT title (#PCDATA)>      
•<!ELEMENT subject (#PCDATA)>       
•<!ELEMENT author (#PCDATA)>      
•<!ELEMENT description (#PCDATA)>      
•<!ELEMENT source (#PCDATA)>      
•<!ELEMENT archive (#PCDATA)>      
•<!ELEMENT offset (#PCDATA)>     
• <!ELEMENT sourceurl (#PCDATA)>       
•<!ELEMENT electronicurl (#PCDATA)>
•<!ELEMENT thumbnailurl (#PCDATA)>
•<!ELEMENT collection (#PCDATA)>
•<!ELEMENT content (#PCDATA)>      
•<!ELEMENT date (#PCDATA)>
This is our XML dtd.  Some of the highlights are the description element which is pulled directly from the record, if there is no description from the page, for instance if it came from a web page, we use a summry or a text snippet.  The archive element indicates the location of the archive on the disk, as I mentioned earlier we are using heretrix to create compressed archives of our content, this element indicates where the archive is located. The offset element indicates the location within the archive of a particular piece of data.  The sourceurl is the link ot the original location of the record, for instance the catalog record.  The electronicurl will link directly to the content.  The content element is there in case some data we gather is in plain text.  We can simply place it in the wrapper.
