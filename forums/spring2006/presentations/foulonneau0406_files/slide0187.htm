Some lessons the CIC experience
nSplitting repositories into sub-collections to enrich an item-level digital library is promising
nData providers’ division of content is in many cases relevant
nData providers tend to define collections when they share content – it is easier to ask for a collection level description at that moment of the workflow
n
nUsing sets to identify those collections is an easy and convenient mechanism vs use of DC relation for eg.
n
nUsing set descriptions to convey CLD is potentially problematic
nThey include information about the set for selective harvesting
nDo not have mechanisms for metadata sharing (eg. datestamp)
