Goals
  • Automation of the digitization process
  • Integrating the actual content and metadata of varieties of object types into one homogeneous repository
  • Preservation and archiving of digital media produced by the Digital Lab or acquired by the library in digital format
  • Enhancing the interoperability and seamless access to the library digital assets

Digital Laboratory
  • State of the art technologies for digitizing
    • Slides in multi-formats and negatives;
    • Books;
    • Manuscripts;
    • Pictures and maps;
    • Audio and video.

  • Staff:
    •  45 staff members recruited and trained
    • Working 7 days a week, two shifts per day

Technology
  • The system is implemented in C# using the Microsoft .Net technology.
  • Web-based components are implemented as ASPX pages running on Microsoft IIS Web server.
  • Repository APIs are implemented as Web services.
  • SQL sever database is used as the main repository database.
  • The repository is integrated with the Virtua ILS which uses Oracle database on Linux platform.

DAK - Metadata
  • Descriptive Metadata
    • For textual materials bibliographic metadata retrieved from the  BA ILS where books are catalogued in MARC21.
    • For multimedia objects VRA fields (Title, Creator(s), Date(s), mateial, technique, Keywords, Description, Dimensions, Location, etc)
  • Administrative Metadata
    • Information about the batches, which are received from the owner.
    • Record document condition, page count, etc.
    • Allocate the document to a Collection
    • Record of completion of each phase with specialist name and date.
    • Information regarding the intellectual property rights
    • Authentication and authorization data
  • Technical Metadata
    • Digital Content Metadata: image resolution, library font, profile, versions
    • Archiving Metadata
    • Publishing Metadata

DAF - Digital Assets Factory
  • Governs the digitization process of the Library collection at the Digital Lab
  • Goals:
    • Provide a database system to keep track of the digitization process
    • Keep track of digitized materials, unifying the naming conventions and exhaustively checking the produced folders and files for consistency
23
DAF - Digital Assets Factory
  • Goals (contd):
    • Provide timely reports to various levels of management describing the workflow and the current status of a certain asset at the Digital Lab
    • Apply necessary encodings on the scanned materials to be suitable for electronic publishing
    • Manage archiving and retrieval of digitized material

DAF Archiving Module
  • Windows application used by DL operator
  • Managing the archiving on CDs and tapes and generate their barcodes.
  • Checking files and folders consistency
  • Automatic packing of digitized files in zipped format with checksum to detect changes.
  • Generating XML file for an archived folder containing metadata
  • Updating the DAK database by recording the archiving information related to a digital file

DAK Cataloging Module
  • Windows application used by Librarian
  • Enriching repository records by adding metadata
  • Creating new records and uploading corresponding digital files
  • Functionalities
    • Configurable templates tailored to specific types of material such as books, maps and slides
    • Importing metadata from external sources such as the library catalog
37
DAK Cataloging Tool
  • Functionalities (contd)
    • Automatic extraction of technical metadata. For example, extracting the height, width and resolution from a JPG file.
    • Batch uploading of files
    • Reviewing tools to display the recently catalogued records and the updates in any controlled list of values
