The Perspective of Three Students
Background knowledge, technical knowledge, library knowledge
Background knowledge from core MLS coursework: user-centered thinking, knowledge organization theory, personnel management issues, etc.

Technical Knowledge
1 semester lecture/lab format class (or three separate workshops) devoted to digital objects: images, text, sound/video. The lecture would present things like terminology, file formats, standards and best practices, and quality control issues, while the lab would offer hands-on experience with the concepts presented in class, including text markup and benchmarking for digital capture.

"1 semester lecture class devoted..."
1 semester lecture class devoted to other technical issues as related to digital libraries: database design, network delivery, large-scale data storage and backup, long-term data migration, methods of searching disparate collections, user-centered interface design, etc.
These areas all need their own experts in a DL program, and the MLS grad won't be it. But the DL-involved librarian will need a general understanding of these concepts and how they apply to DL work.

Specialized Knowledge
1 semester lecture class devoted to metadata formats, standards, implementations, and “crosswalk” methods.
1 semester management-centered class. Mostly lecture for topics like grant-writing, copyright issues, and archival organization, but also some hands-on lab time for preservation considerations.

Application of Knowledge
Part-time work in some sort of DL field would be strongly encouraged while completing coursework.
After the above coursework is completed, an internship would be completed involving managing a small-scale project or planning a pilot for a larger project. This workplace experience is essential in order to tie all the coursework into real-word applications. The intern would also learn about things that are hard to teach in a class -- workflow issues, organizational politics, etc.
