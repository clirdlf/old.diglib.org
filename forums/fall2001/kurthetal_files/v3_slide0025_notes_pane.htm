How can we learn from this in ENCompass?  Unlike OCLC's WorldCat, we are not dealing with MARC records alone nor “things” of the same granularity.  We are adding records in varying metadata element sets, as Marty mentioned earlier.  In MARC, we could conceivably pull format information from the Leader. Where would we do this for non-MARC records?  We would need to examine the feasibility of pulling metadata from certain fields of either the repository, container or object record, depending on the makeup of the collection.  More homogenous repositories would allow for this kind of treatment better.  As an initial step, we could at least display format for MARC records, thereby adding a sort of third dimension to the search results screen.

Displaying format may be one means of managing heterogeneous collections and thus breaking out of flatland in future, but in the meantime, we are focusing on other display techniques.  Marty showed you the relationship manager in the ENCompass staff client a few minutes ago.  You saw it uses an expandable structure and resembles applications like Windows Explorer.  We can click on recognizable icons like file folders to open and reveal logically arranged contents which in turn can be expanded further.
