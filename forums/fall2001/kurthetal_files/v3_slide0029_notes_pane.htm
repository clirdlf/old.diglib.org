Whether or not we agree that different types of users care about hierarchy to varying degrees, we might postulate that all are interested in what type of “thing” they are viewing.  Whereas “traditional” library users were more concerned with whether they were viewing a book or an article, now users are dealing with differences like sound versus graphic versus text file.  They may need to know what application is required to open an object, or how large the object is.  In an environment like ENCompass, where users are searching heterogeneous collections, one way we can provide clarity is to add visual cues for formats of “things,” or objects.

Currently in ENCompass,the only way to ascertain format is through metadata in the object record.  Depending on how the record is set up, the format field or the description field might have text indicating we are looking at a sound file or a text file, etc.  But there is no systematic, prominent mechanism in place to display format type.
