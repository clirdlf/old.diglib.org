Key System Elements
•Collection and repository structures are distinct
•Object records are largely malleable
•Collection structure offers hierarchy
•Searching/browsing interface has potential as a communications tool
    ~ Cornell University Library ~
Working with ENCompass to bring collections from this complex environment to our users, four system elements emerge as important.  Sarah and I will explore them in some detail.
First, the repository structure into which we import resource descriptions and their associated resource links lives separately in the system from the collection structure which organizes resources for presentation to users.
Second, because object record displays are controlled by XSL, we have a relatively high degree of control over them.
Third, the collection structure is hierarchical, which offers a means for establishing relationships among digital objects.
Finally, in addition to serving as a path to lead users to digital objects, the system’s searching-and-browsing interface has the capacity to serve as a communications tool to tell users about digital objects and the relationships among them.  We expect that this capacity of the user interface will be significantly enhanced in the next version of the ENCompass software, Version 2.0, when the search-and-browse interface is driven by XSL as object record displays are now.
