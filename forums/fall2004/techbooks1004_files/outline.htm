"XML builds an infrastructure that..."
  • XML builds an infrastructure that is media-neutral
  • XML separates format from content
    • Begin to think of managing content instead of managing the product
  • XML builds a cross-media publishing infrastructure
    • XML is not a technology, it is a tool, a markup language
    • It is a means to an end
    • It is not a silver bullet
    • XML is simply a guiding principle that will help us get to cross-media publishing
  • Markup accuracy is very critical
    • Automation of tagging using scripts
    • Separation of data capture from markup
    • Understanding of DTD, Schema and content being tagged necessary

"QA of content very necessary"
  • QA of content very necessary
    • Accuracy of data capture using mathematical techniques
    • Consistency of markup
    • Use of automation and technology
    • Multiple parsing engines
    • Need for content checkers
    • Can add cost if not managed correctly
  • Use of Web-based tools for QA (PowerQA)
    • XML based rules for checking content
    • Verifying file names, cross-referencing within and to external objects
    • Parsing of content
    • Visual rendering using XSL/T
    • Automated workflow management along with tracking & feedback

"Need to be integrated into..."
  • Need to be integrated into workflows up-stream
    • Early metadata harvesting
    • Accurate, reliable and digital feeds for ONIX while content in production
    • Libraries can have ‘virtual books’ for approval purposes
    • MARC compatible records at an early stage

    • Building cross-links and cross-references before page proofs
    • Use of multiple name spaces and roles attributes to add value in content
    • Delivery of digital content at multiple stages during production
    • Enforces standardization in publishing process
