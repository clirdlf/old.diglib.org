Why are Digital Formats Important?
  • Interpretation of otherwise opaque content streams is dependent upon knowledge of how typed content is represented
  • Without this knowledge, we can ensure only the fixity of content streams, not their future interpretability and usability
3
Format-specific Operations
  • Identification
    • I have a digital object, what format is it?
  • Validation
    • I have an object purportedly of format F; is it?
  • Characterization
    • I have an object of format F; what are its significant properties?

Format Granularity
  • Identification / validation of general format vs. specific format profile
    • TIFF ¬ TIFF/IT ¬ TIFF/IT/CT ¬ TIFF/IT/CT/P2
  • Variant profiles may require different workflows
    • E.g., compressed vs. uncompressed

  • RLG Automatic Exposure

JHOVE
  • Extensible framework for format-specific:
    • Identification
    • Validation
      • Well-formedness
      • Validity
      • Consistency
    • Characterization
  • Plug-in architecture
  • <http://hul.harvard.edu/jhove/>

Supported Formats
  • ASCII
  • UTF-8
  • TIFF
    • Version 4.0  5.0
    • TIFF/EP (ISO 12234-2:2001)
    • TIFF/IT    (ISO 12639:2003)
      • Type CT, LW, HC, MP, BP, BL, SD, FP; conformance level P1, P2
    • GeoTIFF
    • Exif (JEITA CP-3451)
  • PDF
    • Version 1.0  1.4
    • PDF/X-1a, -2, -3 (ISO 15930:2003)
    • PDF/A (ISO/CD 19005)

18
Representation Information
  • Format-specific
    • ASCII
      • Line endings, control characters
    • UTF-8
      • Number of characters, Unicode character blocks
    • TIFF
      • Byte order, IFD offsets, all tags
      • NISO Z39.87 (in MIX if XML output handler)
    • PDF
      • Images, fonts, annotations, outlines (bookmarks)
      • Filters, document catalog and info dictionary
      • XMP metadata
