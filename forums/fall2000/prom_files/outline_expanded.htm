Evaluation (1)
Strengths
EAD finding aid is standards compliant
Item level description: an imperfect solution for an imperfect world (but archival)
Cost effective: Quick and dirty processing routine
Integration of description into finding aid preserves the full context in which papers found

Evaluation (2)
Strengths (cont.)
Easy to implement elsewhere
tools are easily transportable and available
Integrates to EAD Cookbook (emerging standard)
redefine variables and revise stylesheets
Easily navigable
flexibility for data reuse and indexing
easy reformatting for display (via CSS and Perl)
Next-generation searchable (thru stylesheet)

Evaluation (3)
Weaknesses
Full archival context missing due to copyright restrictions (but similar to other projects)
Caution re: accuracy of description; does not fully comply to rules of archival description
Journalism students
supervision difficult
Length of finding aid (load times)
Server space issues?
